# Global instructions

## Git commits

- Do NOT run `git commit` unless the user explicitly instructs it (e.g. invoking the `/commit` skill, or saying "commit this"). By default, committing is the human's job — stop after staging or after presenting a diff/summary.
- When explicitly asked to commit, follow the `/commit` skill: English Conventional Commits messages, atomic commits, and never add a `Co-Authored-By: Claude` footer.
